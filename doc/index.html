Android SDK & Android Studio {How to setup for working environment?}

Working Environment

Direct testing on real device is the best option, but if you want to install android emulator, install Genymotion (Note: Google Play Service is required to install Google Maps and GC< feature)

1. Install JDK

Need to install Java JDK from here

2. Install Android Studio

Need to install Android Studio as official IDE for Android Development.

3. Install Necessary SDK Package

You can install necessary SDK packages from Android Studio Menu -> Tools -> Android -> SDK Manager
According the app's targetSdkVersion from gradle file, you might need to install Android 9.0 (API 28)

4. Install Emulator

If directly development and testing on real device is the best option but if you want to install android emulator, please install Genymotion (Note : Google Play Service is required to install for Google Map and GCM feature.)

Import Project Into Android Studio {How to import project?}

Import Project

You have to import the project into your android studio and copy into your workspace folder.

1. Where is project?

After you download from Github, there is the project source code at "qrolic/play-hub-app".

2. Import into Android Studio?

You can import from android studio menu (File -> New -> Import Project). When popup menu emerges, go to the folder (mentioned at step 1) and copy that project into your workspace.

3. Make sure the path?

Check SDK and JDK paths for your imported project. To check the paths, right click on project and Open module settings. You can check whether paths are correct or not from emerged popup window.

Rename Package {How to rename the package?}

Rename Package

Currently, my original package name is "com.qrolic.minesweeper" so after you bought, you might need to change according your own package name. For example, if you want to change com.example.app to my.awesome.game.

1. Project pane

In your Project pane, click on the little gear icon

2. Compact Empty Middle Packages

Uncheck / De-select the Compact Empty Middle Packages option

3. Individual Directories

Your package directory will now be broken up in individual directories.

4. Individually select each directory you want to rename

- Right-click it
- Select Refactor
- Click on Rename
- In the Pop-up dialog, click on Rename Package instead of Rename Directory
- Enter the new name and hit Refactor
- Allow a minute to let Android Studio update all changes
- Note: When renaming com in Android Studio, it might give a warning. In such case, select Rename All

5. Gradle Build File

Now open your Gradle Build File (build.gradle - Usually app or mobile). Update the applicationId to your new Package Name and Sync Gradle, if it hasn't already been updated automatically:

Generate Sign apk{How to generate sign apk}


Sign apk

Follow this steps to generate sign apk

Select Build >>> Generat Signed Bundle / APK
Select APK & Click Next
Create New Key Store
1 - KeyStore path :

select a path in your system (Create a path in any drive eg: *F:\AndroidKeys*)give a name to your key(eg:game).save in here with the specific name for your project (F:\AndroidKeys\game.jks).


2 - Password**:

Give some new password and confirm it(Don't forget).


3 - Alias :

Enter an identifying name for your key.


4 - Password :

Create and confirm a secure password for your key. This should be different from the password you chose for your keystore. Validity: select time period for validity of key.


5 - Certificate:

Enter some information about yourslf or organization (like name,..). Done with new key generation.


6 - Click ok
After clicking "OK" on The last step above, it will show you this table
Then Click Next
1 - Choose release
2 - Choose V1 and V2 as a signature Versions
3 - Then Click Finish
